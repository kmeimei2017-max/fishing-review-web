# 린 캔버스 — 선상낚시 후기 게시판

> PRD(`docs/PRD.md`) 기준으로 작성. 검증되지 않은 가설이 포함되어 있으며, 실제 운영 데이터로 주기적으로 갱신 필요.

## 1. 문제 (Problem)

- 선상낚시 후기가 카페/블로그/SNS에 흩어져 있어 특정 선박·포인트·시즌 기준으로 비교하기 어려움
- 후기 작성자는 자신의 글이 노출되고 정리되는 느낌을 받기 어려움 (커뮤니티 피드에 묻힘)
- 기존 대안: 네이버 카페, 블로그, 인스타그램 해시태그

## 2. 고객군 (Customer Segments)

- **작성자**: 선상낚시를 자주 다니는 낚시인 (소셜 로그인으로 직접 후기 작성)
- **방문자**: 배낚시를 계획 중이라 다른 사람 후기로 선박/포인트/시즌을 참고하려는 예비 낚시인
- 얼리어답터: 기존 노션 MVP 단계부터 후기를 남겨온 소수 회원

## 3. 고유 가치 제안 (Unique Value Proposition)

**"흩어진 배낚시 후기를 한곳에, 사진과 함께"**
- 방문자: 로그인 없이 바로 갤러리로 후기를 훑어볼 수 있음
- 작성자: 소셜 로그인 한 번으로 사진과 함께 후기를 남기고, 승인 후 갤러리에 노출됨

## 4. 솔루션 (Solution)

| 문제 | 솔루션 |
| --- | --- |
| 후기가 흩어져 있음 | 하나의 갤러리 페이지(F002)에 모아서 카드형으로 노출 |
| 원하는 후기를 못 찾음 | 제목/작성자 검색(F005) |
| 작성 장벽이 높음 | 소셜 로그인(F006) + 간단한 작성 폼(F007) |
| 품질/스팸 관리 | 관리자 승인 대기열(F008), 승인 전 비공개(F011) |

## 5. 채널 (Channels)

- 검색엔진(SEO) — 후기 상세 페이지 고유 URL(F010)
- 기존 노션 MVP 사용자에게 전환 안내
- 낚시 관련 카페/커뮤니티 공유 (후기 링크 직접 공유)
- *(범위 제외: 카카오톡 공유 버튼 등은 "향후 개선 방향"으로 미룸)*

## 6. 수익원 (Revenue Streams)

- 현재 미정의 — MVP는 무료 공개 게시판으로 수익모델 없음
- 향후 후보(가설 단계, 검증 안 됨): 선박 업체 제휴/광고, 프리미엄 노출

## 7. 비용구조 (Cost Structure)

- Supabase (DB/Auth/Storage) 사용량 기반 비용
- Vercel 배포 비용
- 도메인 비용
- (인건비 제외) 초기 단계는 대부분 인프라 사용량 비용

## 8. 핵심지표 (Key Metrics)

- 누적 승인된(published) 후기 수
- 후기 작성 → 승인 전환율, 승인까지 걸리는 시간
- 갤러리 방문자 수 / 후기 상세 조회수
- 소셜 로그인 신규 가입자 수 (Google/Kakao)

## 9. 경쟁우위 (Unfair Advantage)

- 선상낚시라는 특정 도메인에 집중 (일반 낚시 커뮤니티 대비 좁고 깊은 포지셔닝)
- 관리자 승인 기반 큐레이션으로 스팸/저품질 후기 없는 갤러리 품질 유지
- *(현재는 뚜렷한 해자가 약함 — 검증 필요한 가설)*
