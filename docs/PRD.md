# 선상낚시 후기 게시판 PRD

## 🎯 핵심 정보

**목적**: Supabase를 데이터베이스로 활용하여 회원이 직접 선상낚시 후기를 작성·관리하고, 누구나 웹에서 갤러리 형태로 조회할 수 있는 공개 게시판
**작성자**: 소셜 로그인(카카오/구글 등)한 회원 누구나 — 웹사이트에서 직접 후기 작성, 관리자 승인 후 공개
**방문자**: 누구나 (열람은 로그인 불필요, 후기 작성은 로그인 필요)

## 🚀 아키텍처 전환

이 프로젝트는 노션을 헤드리스 CMS로 쓰는 MVP(1단계)로 시작했으나, 회원가입 기반 웹 후기 작성 기능이 추가되면서 **2단계(Supabase 전환)로 조기 진입**했다.

- **1단계 (완료)**: 노션을 헤드리스 CMS로 사용 — 관리자만 노션에서 작성, 방문자는 읽기 전용. 기능 검증·UX 학습 목적으로는 충분했지만 회원 쓰기 트래픽을 감당할 구조가 아니었음
- **2단계 (진행 중)**: **Supabase(PostgreSQL + Auth)로 전환** — 이유는 두 가지. (1) 노션 API는 rate limit(초당 약 3요청)이 있어 다수 회원이 동시에 글을 쓰는 트래픽을 감당할 수 없음. (2) 노션 API 자체에는 회원 인증/권한 개념이 없어, 소셜 로그인 기반 작성 기능을 구현할 수 없음
- **전환 원칙 (유지)**: 데이터 접근 로직은 반드시 `src/lib/services/review.service.ts` 같은 서비스 레이어 안에만 캡슐화한다. 페이지/컴포넌트가 `@notionhq/client`나 `@supabase/supabase-js`를 직접 호출하지 않도록 하여, 저장소 구현이 바뀌어도 UI/페이지 코드는 최대한 그대로 유지되도록 함
- **기존 노션 데이터**: 이미 노션에 작성된 후기가 있다면 Supabase로 별도 마이그레이션이 필요함 (이 문서는 목표 상태만 정의하며, 마이그레이션 스크립트는 별도 작업)

## 🚶 사용자 여정

### 후기 작성자 (회원)

```
1. 소셜 로그인(카카오/구글 등)으로 회원가입 또는 로그인

2. 후기 작성 페이지에서 제목 + 내용 + 사진 입력 후 제출
   ↓ 제출 즉시 "승인 대기(pending)" 상태로 저장

3. 관리자 승인 대기
   ↓ 관리자가 검토 후 승인하면 "공개(published)"로 전환

4. 완료 (승인 전에는 작성자 본인 외 아무에게도 노출되지 않음)
```

### 방문자

```
1. 웹사이트 홈페이지 접속 (로그인 불필요)
   ↓ 공개(published) 상태인 후기 갤러리 목록 확인

2. 검색으로 원하는 후기 찾기
   ↓ 제목/작성자 검색

3. 후기 카드 클릭 → 상세 페이지
   ↓ 사진 갤러리 확인 (클릭 시 확대)

4. 완료 (후기를 직접 쓰고 싶으면 로그인 필요)
```

## ⚡ 기능 명세

### 1. 핵심 기능

| ID       | 기능명               | 설명                                                   | 관련 페이지                            |
| -------- | -------------------- | ------------------------------------------------------ | -------------------------------------- |
| **F001** | Supabase 데이터 연동 | Supabase(PostgreSQL)를 통해 후기 데이터 조회/저장      | 전체                                   |
| **F002** | 후기 갤러리 목록     | 공개(published)된 전체 후기를 카드 그리드로 조회       | 홈페이지                               |
| **F003** | 후기 상세 조회       | 고유 URL로 특정 후기 내용 표시                         | 후기 상세 페이지                       |
| **F004** | 사진 갤러리          | 후기별 여러 장 사진을 그리드 + 확대(라이트박스)로 표시 | 후기 상세 페이지                       |
| **F005** | 후기 검색            | 제목/작성자 검색                                       | 홈페이지                               |
| **F006** | 소셜 로그인          | 카카오/구글 등 OAuth 기반 회원가입 및 로그인           | 전체                                   |
| **F007** | 후기 작성 폼         | 로그인한 회원이 제목/내용/사진을 입력해 후기 제출      | 후기 작성 페이지                       |
| **F008** | 관리자 승인 대기열   | 제출된 후기를 관리자가 검토 후 공개 여부 결정          | (Supabase 대시보드 또는 관리자 페이지) |

### 2. 필수 지원 기능

| ID       | 기능명          | 설명                                                                                                           |
| -------- | --------------- | -------------------------------------------------------------------------------------------------------------- |
| **F010** | 후기 URL 생성   | 후기 레코드 ID 기반 고유 URL 생성 (`/review/[id]`)                                                             |
| **F011** | 공개 범위 제어  | `status`가 `published`가 아닌(= `pending`) 후기는 목록/상세 어디에도 노출되지 않음 (서버에서 항상 강제 필터링) |
| **F012** | 유효성 검증     | 존재하지 않거나 비공개(pending)인 후기 접근 시 404 에러 처리                                                   |
| **F013** | 반응형 레이아웃 | 모바일/태블릿/데스크톱 대응                                                                                    |
| **F014** | 다크모드        | 라이트/다크/시스템 테마 지원                                                                                   |

## 🗄️ 데이터 모델

### Supabase 테이블

**`reviews` 테이블**

| 필드       | 타입      | 설명                                                |
| ---------- | --------- | --------------------------------------------------- |
| id         | uuid      | 후기 고유 ID (기본키)                               |
| title      | text      | 후기 제목                                           |
| author_id  | uuid      | 작성자 (FK → `auth.users.id`)                       |
| images     | text[]    | 사진 URL 목록 (Supabase Storage)                    |
| content    | text      | 후기 내용                                           |
| status     | text      | 공개 상태: `pending`(승인 대기) / `published`(공개) |
| created_at | timestamp | 작성일시 (자동)                                     |

**회원 정보**: Supabase Auth가 관리하는 `auth.users` 테이블을 그대로 사용 (소셜 로그인 provider 정보 포함).

## 🛠️ 기술 스택

### 프론트엔드 프레임워크

- **Next.js 15.5.20** (App Router) - React 풀스택 프레임워크
- **TypeScript 5.x** - 타입 안전성 보장
- **React 19** - UI 라이브러리

### 스타일링 & UI

- **TailwindCSS v4** - 유틸리티 CSS 프레임워크
- **shadcn/ui** - 고품질 React 컴포넌트 라이브러리
- **Lucide React** - 아이콘 라이브러리

### 백엔드 & 인증

- **Supabase** - PostgreSQL 데이터베이스 + Auth(소셜 로그인) + Storage(사진 업로드)
- **@notionhq/client** - 1단계(레거시) 노션 연동에 사용되었던 SDK. 마이그레이션 완료 후 제거 예정

### 배포 & 호스팅

- **Vercel** - Next.js 15 최적화 배포 플랫폼
- **환경 변수**: `NEXT_PUBLIC_SUPABASE_URL`, `NEXT_PUBLIC_SUPABASE_ANON_KEY`, `NEXT_PUBLIC_BASE_URL` (관리자 승인은 Supabase 대시보드에서 처리하므로 Service Role Key는 앱에서 사용하지 않음)

## 📦 핵심 구현 로직 (목표 형태)

### 1. 후기 상세 조회 (Server Component)

```typescript
// app/review/[id]/page.tsx
import { getOptimizedReview } from '@/lib/services/review.service'

export default async function ReviewPage({ params }: ReviewPageProps) {
  const { id } = await params
  const review = await getOptimizedReview(id) // status가 'published'가 아니면 에러 → notFound()
  return <ReviewHeader review={review} />
}
```

### 2. 갤러리 목록 조회

```typescript
// app/page.tsx
import { getPublishedReviews } from '@/lib/services/review.service'

const { reviews, nextCursor, hasMore } = await getPublishedReviews(12)
```

### 3. 후기 작성 (로그인 필요)

```typescript
// app/write/page.tsx (Server Action)
import { createReview } from '@/lib/services/review.service'

// 로그인한 사용자만 접근 가능, 제출 시 status: 'pending'으로 저장
```

### 4. 고유 URL 구조

```
https://yourdomain.com/review/[reviewId]
```

## ✅ 성공 기준

1. ✅ 방문자는 로그인 없이 공개(published)된 후기 전체를 갤러리로 확인 가능
2. ✅ 회원은 소셜 로그인 후 후기를 작성/제출할 수 있음
3. ✅ 제출된 후기는 관리자 승인 전까지 목록/상세 어디에도 노출되지 않음
4. ✅ 관리자 승인 후에만 후기가 공개 갤러리에 노출됨
5. ✅ 고유 URL로 접근 시 후기 상세와 사진 갤러리가 정확하게 표시됨
6. ✅ 모바일/태블릿/데스크톱에서 정상 작동
7. ✅ 잘못된 URL 또는 비공개 후기 접근 시 적절한 에러 메시지 표시
