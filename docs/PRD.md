# 노션 기반 선상낚시 후기 게시판 PRD

## 🎯 핵심 정보

**목적**: 노션을 데이터베이스로 활용하여 선상낚시 후기를 관리하고, 누구나 웹에서 갤러리 형태로 조회할 수 있는 공개 게시판
**작성자**: 관리자(운영자) — 노션에서 직접 후기 작성
**방문자**: 누구나 (로그인 불필요, 전체 후기 갤러리 열람)

## 🚀 스케일업 로드맵 (중요)

**최종 목표: 수십~수백만 명 규모의 후기 게시판**. 현재 노션 연동은 MVP/학습 단계이며, 영구적인 구조가 아님.

- **1단계 (현재)**: 노션을 헤드리스 CMS로 사용 — 기능 검증, UX 학습, 소규모(동호회~수천 명) 운영에 적합
- **2단계 (스케일업)**: 회원 규모가 커지면 **Supabase(PostgreSQL) 등 자체 DB로 전환** — 노션 API의 rate limit(초당 약 3요청), 제한적인 쿼리 능력, 단일 장애점 문제를 해결하기 위함
- **전환을 대비한 설계 원칙**: 노션 관련 로직은 반드시 `src/lib/services/review.service.ts` 같은 서비스 레이어 안에만 캡슐화할 것. 페이지/컴포넌트가 `@notionhq/client`를 직접 호출하지 않도록 하여, 나중에 서비스 레이어 내부 구현만 Supabase 쿼리로 교체해도 UI/페이지 코드는 거의 그대로 유지되도록 함

## 🚶 사용자 여정

### 후기 작성자 (관리자)

```
1. 노션 데이터베이스
   ↓ 후기 내용 + 사진 입력, "공개 여부"는 "비공개(작성중)"로 유지

2. 작성 완료 후 "공개 여부"를 "공개"로 변경
   ↓ 최대 60초 이내 웹사이트에 자동 반영

3. 완료 (별도 발행 절차 없음)
```

### 방문자

```
1. 웹사이트 홈페이지 접속
   ↓ 전체 후기 갤러리 목록 확인

2. 검색으로 원하는 후기 찾기
   ↓ 제목/작성자 검색

3. 후기 카드 클릭 → 상세 페이지
   ↓ 사진 갤러리 확인 (클릭 시 확대)

4. 완료
```

## ⚡ 기능 명세

### 1. 핵심 기능

| ID       | 기능명                 | 설명                                                   | 관련 페이지      |
| -------- | ---------------------- | ------------------------------------------------------ | ---------------- |
| **F001** | 노션 데이터베이스 연동 | Notion API를 통해 후기 데이터 조회                     | 전체             |
| **F002** | 후기 갤러리 목록       | 공개된 전체 후기를 카드 그리드로 조회                  | 홈페이지         |
| **F003** | 후기 상세 조회         | 고유 URL로 특정 후기 내용 표시                         | 후기 상세 페이지 |
| **F004** | 사진 갤러리            | 후기별 여러 장 사진을 그리드 + 확대(라이트박스)로 표시 | 후기 상세 페이지 |
| **F005** | 후기 검색              | 제목/작성자 검색                                       | 홈페이지         |

### 2. 필수 지원 기능

| ID       | 기능명          | 설명                                                                                            |
| -------- | --------------- | ----------------------------------------------------------------------------------------------- |
| **F010** | 후기 URL 생성   | 노션 페이지 ID 기반 고유 URL 생성 (`/review/[id]`)                                              |
| **F011** | 공개 범위 제어  | "공개 여부"가 "공개"가 아닌 후기는 목록/상세 어디에도 노출되지 않음 (서버에서 항상 강제 필터링) |
| **F012** | 유효성 검증     | 존재하지 않거나 비공개인 후기 접근 시 404 에러 처리                                             |
| **F013** | 반응형 레이아웃 | 모바일/태블릿/데스크톱 대응                                                                     |
| **F014** | 다크모드        | 라이트/다크/시스템 테마 지원                                                                    |

## 🗄️ 데이터 모델

### Notion Database (후기 정보)

| 필드       | Notion 속성명 | 타입           | 설명                        |
| ---------- | ------------- | -------------- | --------------------------- |
| id         | (자동)        | Notion Page ID | 후기 고유 ID                |
| title      | 후기 제목     | Title          | 예: "거문도 참돔 조황 대박" |
| author     | 작성자        | Rich text      | 표시용 닉네임               |
| images     | 사진          | Files & media  | 여러 장 업로드              |
| content    | 후기 내용     | Rich text      | 본문                        |
| visibility | 공개 여부     | Select         | "공개" / "비공개(작성중)"   |

노션 속성명은 코드(`src/types/notion.ts`)와 1:1로 매칭되므로, 실제 데이터베이스를 만들 때 위 이름을 정확히 그대로 사용해야 합니다. 자세한 작성 방법은 `docs/notion-guide.md` 참조.

### 외부 API & 통합

- **@notionhq/client** - Notion API 공식 SDK

### 배포 & 호스팅

## 🔑 Notion API 설정 가이드

`docs/notion-guide.md` 문서에 상세 절차가 있습니다. 요약:

1. [Notion Developers](https://www.notion.so/my-integrations)에서 Integration 생성 → 토큰을 `.env.local`에 저장
2. 후기 데이터베이스 생성 (위 데이터 모델 표대로 속성 설정)
3. 데이터베이스 "..." → "Add connections" → Integration 연결
4. 데이터베이스 ID를 `.env.local`에 저장

## 📦 핵심 구현 로직

### 1. Notion 데이터 조회 (Server Component)

```typescript
// app/review/[id]/page.tsx
import { getOptimizedReview } from '@/lib/services/review.service'

export default async function ReviewPage({ params }: ReviewPageProps) {
  const { id } = await params
  const review = await getOptimizedReview(id) // "공개" 상태가 아니면 에러 → notFound()
  return <ReviewHeader review={review} />
}
```

### 2. 갤러리 목록 조회

```typescript
// app/page.tsx
import { getReviewsFromNotion } from '@/lib/services/review.service'

const { reviews, nextCursor, hasMore } = await getReviewsFromNotion(12)
```

### 3. 고유 URL 구조

```
https://yourdomain.com/review/[notionPageId]
예: https://fishing-review.example.com/review/abc123def456
```

## ✅ 성공 기준

1. ✅ 노션 데이터베이스에서 후기 정보를 정상적으로 가져옴
2. ✅ 홈페이지에서 공개된 후기 전체를 갤러리로 확인 가능
3. ✅ 고유 URL로 접근 시 후기 상세와 사진 갤러리가 정확하게 표시됨
4. ✅ "비공개(작성중)" 후기는 목록/상세 어디에도 노출되지 않음
5. ✅ 모바일/태블릿/데스크톱에서 정상 작동
6. ✅ 잘못된 URL 접근 시 적절한 에러 메시지 표시
